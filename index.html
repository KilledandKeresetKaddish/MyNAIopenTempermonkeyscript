TagLab 一个给女性向 / 男性刻画使用者的 NovelAI 画师 tag 实验库

这个网站最初是为了和 nax.moe 那类"画女孩的实验库"对称存在——以 男性为实验对象, 把 danbooru 画师 tag 的产出按维度归档,方便在生图时组合出合适的画师串。 做着做着长出了评分、收藏夹、调色板、Prompt 解析和一堆别的东西。

状态个人站 传播Discord /小圈 客服没有 本页目的减少被问
§ 01

这是什么 / 不是什么

先劝退,再说怎么用。

是什么

  • 按 Dimension(维度) 组织的画师 / 标签实验库——每个维度固定一套 prompt,跑出来的图可以横向比较画师
  • 你可以 搜索、筛选、评分、收藏、复制画师名、一键跳 danbooru
  • 登录到一定级别后可以 在线用内嵌 NAI 调色板出图、导入 PNG/WebP 解析 prompt、管理 串串包

不是什么

  • 不是 NovelAI 使用入门。你需要已经会用 NAI 或类似工具
  • 不是大众化产品,没有客服,没有工单,没有 roadmap
  • 不是对所有人友好。部分维度和功能需要级别
  • 不是对所有人客气。站长中文很差英文也很差法语也很差日语也很差, 但是脾气更差。
18+ 警告。部分维度 / 功能涉及 NSFW 内容,未成年请勿进入。
§ 02

维度(Dimension)

每个维度 = 一套固定的实验 prompt + 该 prompt 下的所有画师实验图。同一画师可能出现在多个维度,但评分彼此独立。

主要维度一览

Loose prompt for male 公开
最主力的维度。提示词很少,只有 画师 tag + 1boy 再加点 Heavy & Quality。大多数情况下你看这里就够了。
Constrain prompt for male 🔒 登录
严格约束的版本:限定发色、瞳色、简单背景、cowboy_shot。用来看脸和涩度,方便横向比。
Face Tag for Male 公开
针对眼睛 / 面部 tag 的实验维度。不再是画师 subject,而是标签subject例如:smile。
hair tag on male 公开
头发 tag 实验。同上。
Copyright Tag on Male 🔒 登录
IP 作品 tag 的男性化实验。
Nsfw Duo Hetro Sex Prompt 🔒 18+ 登录
涩涩维度。摆明了写着 NO AGE UNDER 18。你进来看见做爱瞎眼我能怎么办?
Test dimension和🚮待分类维度 占位
内部测试入口,摆设用。忽略。
Invalid / 已测试太丑 公开
无效 tag 归档处。不生图,纯记录"测过但不行"。
怎么选?平时找画师 → Loose。想看脸 / 看涩度 / 想横向比 → Constrain/NSFW。其它维度是想挖 tag 的时候再去。未来还会一拍大腿/亲友胡扯而添加。

§ 04

搜索语法

搜索框支持一套 mini 查询语言。下面是全部可用写法——点击搜索框左边的"?"也能在站内看到同样的 cheatsheet。

// 1) 普通关键词 —— 会匹配 tag 名 / labels
女性向

// 2) 只检索 label
label:女性向        // 不检索画师名

// 3) 只检索我的评论(需登录)
personal:女性向

// 4) 逻辑运算
厚涂 AND 一般向
平涂 OR 厚涂
-厚涂              // 排除
NOT 厚涂

// 5) fav收藏筛选
is:fav              // 任意 fav1/2/3
is:not              // 我没收藏的
is:fav1 / fav2 / fav3
fav1                // 简写也可

// 6) 评分条件
♥>3  ★<1        // 全局评分
personal:♥>3  personal:★>2   // 个人评分

// 7) 评论检索
comment:女性向   // 检索所有人的评论

// 8) 收藏夹筛选
folder:名称          // 标签匹配名称的文件夹
folder:*             // 任一文件夹里的标签
-folder:$            // 不在任何文件夹

// 9) 投票
is:upvoted / is:downvoted
votes>3  votes<0       // 全站净票数

// 10) 组合
label:女性向 AND -厚涂 AND is:fav2
(男性向 AND personal:♥<2) OR ♥>0
Tip. 搜索框下方的快捷 chip,是给懒得打字的快捷键,不是完整的语法。要灵活组合还是老老实实手写。

卡片底部的彩色分割线

分割线不是装饰。它代表同名画师在其它维度的交互状态:已评分/已评论/已收藏....透明=该subject其他维度无记录。

评分系统的设计意图

♥ Heart

出图的男性化程度。越像个"男人"越高分。纯为个人感受,不必纠结。

★ Star

整体风格好不好。画面质量、有没有想用的感觉。

↑ / ↓ YES/NO

粗暴的二元票。想保存 / 想屏蔽都可以用它。

以上只是初始设定的含义。实际上随你打,没人会查。只是自己横向比的时候分母一致会比较好用。但你对个撩衣服高潮脸伪娘打♥=10, 我看见了会质疑你的女性身份/性取向。

§ 07

收藏夹系统

两套并行:固定的 fav1 / fav2 / fav3 快速收藏位,以及自建的 文件夹 (folders)。

fav1 / fav2 / fav3

三个固定槽位,卡片上一键勾选。你可以自己定义含义,比如 fav1=必用、fav2=观察中、fav3=涩涩专用。搜索语法里用 is:fav1 等检索。

文件夹 (folders)

比 fav 自由。卡片上点一下文件夹图标,在浮层里勾选要加入的文件夹或新建一个。详情页也能批量管理。

🔍 搜索里怎么用。

  • folder:冬天的参考 —— 该文件夹里的
  • folder:* —— 在任一文件夹里
  • -folder:$ —— 不在任何文件夹里
bypasser 级别用不了收藏夹。要用请先升到 visitor 或以上。见 §12。

关键概念

标签页
一个调色板可以有多个"展示区 / 分组",每个是一组画师串的子集。
导出的提示词

::{p}:: 占位符
在 Prompt 预览里写 ::{p}::,生图时自动替换成上面那串,方便把画师串塞进完整 prompt 的任意位置。
NAI API Key
仅本地存。作者不会也没法回收/信任——不要指望管理员帮你找。
画风实验
一键用当前串跑几张图,快速判断画师组合是否可行。
保存 / 读取模板
常用组合可存为模板,避免每次重拼。
Character
最多 6 个 character prompt/uc,角色级权重独立。
转移到串串包
把当前调色板沉淀成串串包里的一个条目。
数据丢失自负。此功能为💣某姓A名nan💣的家伙特供想法, 因此暂不广泛开放省的有人说我偷你api好吧?信息存服务器但作者不会管。不要放重要或唯一的prompt / Key。请及时自行备份(导出调色板)。

§ 10

Prompt 导入

把一张 NAI 生成的 PNG / WebP 拖进去,解析出它的 prompt / seed / model 等信息,并标红它用了哪些画师、以及这些画师是否在本站有归档。

支持的格式

  • PNG:tEXt chunks + stealth LSB
  • WebP:EXIF UserComment

解析后你会看到

元信息
Source · Software · Model · Seed · Steps · Sampler · Scale
正面 / 负面提示词
每个 tag 都是独立 chip,鼠标悬停可以看权重。
画师匹配
识别 artist:xxx,标出该画师在本站的维度。右下角会显示一张 mini 详情卡,支持"点击添加到调色板"。
Danbooru API Key
可选。配了以后可以查 tag 的 danbooru 元信息(类别、别名)。
用途:别人友善发你原图,你经过许可后查看画师串 → 丢进来 → 看 artist: 部分 → 直接点自动链接的卡片。
§ 11

个人主页

右上角用户名点进来。汇总你在每个维度下的所有评分、收藏、评论。

账号密码
默认 无密码登录(直接输入用户名就能进)。如果你想设密码,在这里设。
第一 / 二 / 三排序
三层级联排序:比如先按 ♥ 高→低,再按 ★ 高→低,再按 收藏 状态。
每个维度一个表
Loose / Constrain / Face / Hair ... 每个维度一张表,列出你交互过的 subject,含预览、♥、★、fav、评论。
个人主页是只读汇总——要修改评分还是得回到具体卡片或详情页。

§ 13

规则与禁区

想让作者保持好心情和你长期共处,请看这里。

关于标签建议

  • 开放"标签建议"是因为作者知道自己归档不完善,不是为了收一大堆要审核的建议
  • 胡乱提交、跟风提交、带情绪提交 → 会被无视或封建议权
  • 提建议前先自己过一眼该 tag 在其它维度的状态,确保确有必要

关于 subject / 画师

  • subject 缺失:直接用"上传 subject"页面提交,附画师名和 danbooru 链接
  • 认为 subject 放错维度:准备好生图方面的证据(出图对比 / 提示词证据),发 DC 小号
  • 纯感觉、纯吐槽的反馈 → 不看

关于扩散

  • 本站是个人向、小圈 DC 传播。公开传播网址你自己负责当客服哈
  • 不要把本站当 NovelAI 教学入口。这里假设你已经会用

关于 NSFW

  • 未成年禁止进入
  • NSFW 维度默认不向 bypasser / visitor 打开
  • 不要说是从本站来的
数据不保证。本站是作者个人项目,可能随时丢数据、改接口。但不会说跑路就跑路, 只会在作者被攻击/不想续服务器然后到期后自然下线消散。
§ 14

常见问题

来找作者之前请先看完这一节。

Loose / Constrain / Face / Hair 有同样的画师,评分能不能连通?
不能。每个维度的评分独立。但卡片下方的彩色分割线会告诉你"这个人你在别处也交互过";详情页右侧有一键跳同名不同维度的按钮。
手机上那个刷卡模式是什么?
自己研究。作者不解释。移动端除了 画廊主页 / 详情页 / 收藏夹 之外都不做适配(调色板、串串包、Prompt 导入、subject 提交等在手机上不保证可用)。
我觉得这个 tag 不对 / 这个画师放错维度了,怎么办?
· Tag → 在详情页用"标签建议"提交,管理会看。
· Subject 缺失 → 用"上传 subject"页面直接提交。
· Subject 放错维度 → 准备生图证据,发 DC 小号。纯吐槽不看。
